Detailed Description

z-order transform

Potential optimization

  • get_strides ends up computing the same values over and over again... that is the get_bit(i)*strides[i] is always the same...
    See also:
    DomainList
    • for each level there are a d^ndim children
    • log2(shape[0]) levels
    • for each level need
      • the shape
      • the input block strides
      • the output block strides
  • right now the terminal case is when the shape is 1x1x1x...1

However, stopping at the 2x2x2x...2 level, could take advantage of block copies? The size of the block is 2xsizeof(T) so maybe not a big deal.

A more general view of this optimization is to unroll the last few iterations to avoid some excess computation.
