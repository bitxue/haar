Inverse z-order transform.

Parameters:
[in] ndim The number of dimensions. The input and output fields must have the same number of dimensions, but the size of some dimensions may be 1.
[in] shape An array of size ndim. It describes the size of the rectangular volume to copy.
[in,out] out The output data volume.
[in] ostrides An array of size ndim+1 that describes the memory layout of out. ostrides[i] should be the number of items between two adjacent voxels on dimension i. ostrides[ndim] is the total number of bytes in out.
[in] in The input data volume
[in] istrides An array of size ndim+1 that describes the memory layout of in.

Implementation is the same as the forward transform except the role of out and in are swapped.
